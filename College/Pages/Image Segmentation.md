Segmentation subdivides an image into its constituent regions or objects. The process of segmentation should stop when the objects or regions of interest in an application have been detected. For example, in electronics assembly, segmentation should stop at the level in which the smallest component is detected where there are chances of error in assembly.

Components in an image are broadly differentiated using the concepts of **discontinuity** and **similarity**. In discontinuity, the assumption is that boundaries of regions are sufficiently different from each other and from the background to allow boundary detection based on local discontinuities in intensity. **Edge-based segmentation** is the principal approach used in this category. **Region-based segmentation** approaches in the second category are based on partitioning an image into regions that are similar according to a set of predefined criteria.

# Background detection
Since we know that local averaging, implemented through integration, has a smoothening effect in an image, it is logical to believe that derivation leads to detection of abrupt changes in the regions of an image by using spatial filters/masks. In general, we have:

1. First-order derivatives generally produce thicker edges in an image.
2. Second-order derivatives have a stronger response to fine detail, such as thin lines, isolated points, and noise.
3. Second-order derivatives produce a double-edge response at ramp and step transitions in intensity.
4. The sign of the second derivative can be used to determine whether a transition into an edge is from light to dark(-ve) or dark to light(+ve).

# Isolated Point detection
Point detection is based on the second derivative, particularly the Laplacian:
$$
\nabla^2f(x, y) = \frac{\delta^2f}{\delta^2x}+\frac{\delta^2f}{\delta^2y}
$$
which can be simplified to:
$$
\nabla^2f(x, y) = f(x + 1, y) + f(x - 1, y) + f(x, y + 1)
+ f(x, y - 1) - 4f(x, y)
$$
A point is said to be detected in a binary output image if the output of the mask at a specific point exceeds some threshold defined by:

$$
g(x, y) =1if|R(x,y)| >= T
$$
and 0 otherwise. Here, T is the threshold and R is the output of the mask given by:
$$
R = w_1r_1 + w_2r_2 + ... + w_nr_n
$$
n is the dimension of the mask, $r_n$ is the intensity at point with coefficient $w_n$.

The idea of detection is essentially that a point will have an intensity much different than its neighbours, only detected if the differnce is large enough as governed by T.

==Note that, as usual for a derivative mask, the coefficients sum to zero, indicating that the mask response will be zero in areas of constant intensity.==

# Edge Detection
Detecting changes in intensity for the purpose of finding edges can be accomplished using first- or second-order derivatives. Specifically, the **gradient** is givem by:

$$
\nabla f(x,y)=\begin{bmatrix} g_x \\ g_y \end{bmatrix}=\begin{bmatrix} \frac{\delta f}{\delta x} \\ \frac{\delta f}{\delta y}\end{bmatrix}
$$
The gradient points in the direction of **maximum intensity change**. In this regard, edges correspond to **large gradient values**.

## Edge Strength and Direction

The Magnitude or edge strength is given by:
$$M(x,y)= \sqrt{g_x^2 + g_y^2}$$

The above is often approximated for efficiency:

$$M(x,y) \approx |g_x| + |g_y|$$

And the Direction or angle is given by:

$$α(x,y)=\tan^{-1}\left(\frac{g_y}{g_x}\right)$$
It is important to note that the gradient direction is perpendicular to the edge, ie, Edge direction = $\alpha - 90∘$

However, since digital images are discrete in nature, we can make a discrete approximation of the following:
$$g_x​=f(x+1,y)−f(x,y)$$$$g_y=f(x,y+1)−f(x,y)$$
Other gradient operators are:
### Roberts Operator (2×2)

- Uses diagonal differences:

$$g_x=r_9−r_5,g_y=r_8−r_6$$

- Simple but:
    - Sensitive to noise
    - Poor directional accuracy
### Prewitt Operator (3×3)

$$g_x=(r_7+r_8+r_9)−(r_1+r_2+r_3)g_y = (r_3+r_6+r_9) - (r_1+r_4+r_7)$$

- Uses larger neighborhood → better estimation
- Detects horizontal & vertical edges

### Sobel Operator (3×3)

$$g_x=(r_7+2r_8+r_9)−(r_1+2r_2+r_3)g_y = (r_3+2r_6+r_9) - (r_1+2r_4+r_7)$$

- Adds **weight 2 to center**
- Provides **smoothing + edge detection**
- Preferred in practice

## Gradient Image

- Compute $g_x, g_y$ at every pixel
- Combine to form:
    - **Gradient magnitude image → edge map**
    - **Gradient direction image → orientation info**

## Directional Sensitivity

- $g_x$ → emphasizes **horizontal edges**
- $g_y$ → emphasizes **vertical edges**
- Additional masks used for **diagonal edges (±45°)**

## Noise and Smoothing

- Derivatives amplify noise
- Solution:
    - Apply **smoothing (e.g., averaging filter)** before gradient
- Trade-off:
    - Smoothing ↓ noise but also ↓ edge sharpness

## Thresholding

Edge detection is finalized using thresholding:

$$
g(x,y)=\begin{cases} 1 & \text{if } M(x,y) \ge T \\ 0 & \text{otherwise} \end{cases}$$

- High T → fewer, sharper edges (may break edges)
- Low T → more edges (includes noise)

## Combined Approach

Typical pipeline:

1. **Smooth image**
2. **Compute gradient (Sobel/Prewitt)**
3. **Compute magnitude**
4. **Apply threshold**

# Edge Linking and Boundary Detection
Ideally, edge detection algorithms should generate pixels that lie completely on edges. However, due to irregular illumination and noise we get a serious of broken edges with discontinuities. To resolve this, we apply various linking algorithms after edge detection.

## Local processing
The following conditions are checked against each pixel declared an edge point by the algorithm:
1. Magnitude
2. Direction of gradient vector

An edge pixel with coordinates (s, t) is similar in magnitude to the pixel at (x, y) if
$$| M(s, t) - M(x, y)| \le E$$
where E is a positive threshold.

An edge pixel with coordinates (s, t) has an angle similar to the pixel at (x, y) if
$$|\alpha(s, t) - a(x, y)| \le A$$
where A is a positive angle threshold.It must be noted that the direction of the edge at (x, y) is perpendicular to the direction of the gradient vector at that point.

The preceding formulation is computationally expensive because all neighbours of every point have to be examined.

## Regional Processing
Instead of computing the linking of every individual pixel, we can improve the detection speed by considering edge pixels as part of a region boundary and approximate the shape of the region based on some polygon. This technique is adopted as pixels of the same objects are often known beforehand or can be determined.

The following must be satisfied:
1. The points must be ordered to be able to traverse them in clockwise/anticlockwise direction
2. The starting and ending vertices must be known(often the extreme/leftmost or rightmost vertices)

This results in simpler representation and preservation of the region shape.

The procedure is:
- Draw line AB, where A and B are starting vertices
- Compute perpendicular distance of all intermediate points to line AB
- Find point with maximum distance
- If $D_{max} > T$ where T is some threshold, that point becomes a vertex
- Recursively repeat on smaller segments

The threshold value must be carefully chosen to balance noise generation and over-smoothing and approximation of the boundary.

Applications include Medical Imaging, forensics, object recognition and shape representation.

Several pre-processing operations are performed before polygon fitting like gradient-edge detection, thresholding and majority filtering.


## Global processing and Hough transform
Regional processing may be applied when we have some idea of where points of interest must be. When all pixels of an image are candidates for a region, we check them against some global properties.

The **Hough transform** can detect lines, curves etc even in the presence of noise and broken lines. Since direct line detection is computationally expensive(about $O(n^3)$) we use the normal form to express points in the parameter space:
$$\rho = x_i\cos\theta + y_i\sin\theta$$
When such sinusoids intersect at some $(\rho^*,\theta^*)$, we get a common line.

This method is still computationally expensive for large parameter spaces. Applications include road, runways and shape detection.

# Thresholding segmentation

## Primary Idea

Segment an image by **separating pixels based on intensity values**.

- Pixels are classified into regions depending on whether they are **above or below a threshold**.

## Core Formula (Binary Thresholding)
$$
g(x,y) =
\begin{cases}
1 & \text{if } f(x,y) \ge T \\
0 & \text{if } f(x,y) < T
\end{cases}
$$
f(x,y) is the input image, g(x, y) is the segmented output and T is the threshold.

## Multilevel Thresholding

For multiple regions:
$$
g(x,y) =
\begin{cases}
0 & f < T_1 \\
1 & T_1 \le f < T_2 \\
2 & f \ge T_2
\end{cases}
$$

## Types of Thresholding

### 1. Basic Global Thresholding(Iterative Thresholding)
Global Thresholding sets a single threshold for entire image and assumes uniform lighting of the image. Iterative thresholding is a global threshold segmentation method that automatically computes a threshold by repeatedly separating image pixels into two groups and updating the threshold using the mean intensities of those groups.

It begins by choosing a initial threshold, $T_0$. A common value is the average.
$$
T_0 = \frac{1}{2}(z_{\min}+z_{\max})
$$
We then perform foreground and background separation based on the groups:
$$
G_1 = \{(x,y)\mid f(x,y) > T_k\}
$$
and 
$$
G_2 = \{(x,y)\mid f(x,y) \leq T_k\}
$$

we then calculate the mean for each group:
$$
\mu_n = \frac{1}{|G_n|}\sum_{(x,y)\in G_n} f(x,y)
$$
and then update the threshold:
$$
T_{k+1} = \frac{1}{2}(\mu_1+\mu_2)
$$
The stopping condition is applied when the difference in successive thresholds become very small:
$$
|T_{k+1}-T_k| < \epsilon
$$
The final segmentation is applied as:
$$
g(x,y)=
\begin{cases}
1, & f(x,y) > T \\
0, & f(x,y) \leq T
\end{cases}
$$

The advantages of this technique are:
- Simple and easy to implement
- Fully automatic after initialization
- Fast convergence
- Effective for bimodal histograms

and limitations are:

- Sensitive to initial threshold
- Assumes two dominant classes
- Performs poorly under:
    - uneven illumination
    - high noise
    - overlapping intensity distributions


This technique is useful in:

- basic image binarization
- preprocessing before morphology
- document segmentation
- medical preprocessing
- industrial inspection systems


### 2. Local (Adaptive) Thresholding
Here, the threshold varies per pixel which enables the handling of non-uniform illumination.
$$
T = T(x,y)
$$

### 3. Automatic Thresholding (e.g., Otsu)
Otsu’s method is an automatic threshold selection technique used for image segmentation. It involves assume a histogram with two classes- foreground and background, and the optimizing the value of T to best separate these classes.

For a grayscale image with intensity levels 0 to L - 1:

Probability of Background class = $\omega_0(T)=\sum_{i=0}^Tp(i)$

Probability of Foreground class = $\omega_1(T)=\sum_{i=T+1}^{L-1}p(i)$

The between-class invariance is then:
$$
\sigma_b^2(T)
=
\omega_0(T)\omega_1(T)
\left[
\mu_0(T)-\mu_1(T)
\right]^2
$$
Where foreground mean intensity:
$$
\mu_1(T) = \frac{\sum_{i=T+1}^{L-1} i\,p(i)}{\omega_1(T)}
$$
Background mean intensity:

$$
\mu_0(T) = \frac{\sum_{i=0}^{T} i\,p(i)}{\omega_0(T)}
$$
And the optimal threshold is:
$$
T^* = \arg\max_T \sigma_b^2(T)
$$
We then segment the image based on:

$$
g(x, y) = 
\begin{cases} 
      x^2 & \text{if } f(x,y) > T* \\
      0 & \text{if } f(x,y) < T*
\end{cases}
$$

The algorithm is:
- Compute image histogram
- Normalize histogram
- For every possible threshold TTT:
    - compute class probabilities
    - compute class means
    - compute between-class variance
- Select threshold with maximum variance
- Segment image using that threshold

The advantages and limitations of Otsu's methods are:

- Fully automatic
- Simple and fast
- No manual threshold selection
- Works well for bimodal histograms
- Widely used in preprocessing pipelines

but
- Assumes two dominant classes
- Performs poorly when:
    - illumination is uneven
    - histogram is not bimodal
    - foreground/background overlap heavily
    - noise is high

Use cases are:
- Document image binarization
- Medical image preprocessing
- Industrial inspection
- Object-background separation
- OCR systems
- Microscopy image segmentation
## Core Insight

> Thresholding segments an image by **partitioning intensity space into regions**, assuming objects differ in brightness from the background.

## Use Cases

### 1. Simple Object Extraction

- When object and background have distinct intensities
- Example: text on white background

### 2. Preprocessing for Other Methods

- Used before:
    - Morphological operations
    - Watershed segmentation

### 3. Medical Imaging

- Segment tissues with different intensities (e.g., bones vs soft tissue)

### 4. Industrial Inspection

- Detect defects or shapes with clear contrast

## Advantages

- **Simple and fast**
- Low computational cost
- Easy to implement

## Limitations

- Sensitive to:
    - Noise
    - Illumination changes
- Fails when intensity distributions overlap

## One-Line Summary

> Thresholding segmentation classifies pixels based on intensity using a threshold, producing a simple and efficient separation of objects from background.

# Using moving averages
A special case of the local thresholding method just discussed is based on computing a moving average along scan lines of an image. This implementation is quite useful in document processing, where speed is a fundamental requirement. The scanning typically is carried out line by line in a zigzag pattern to reduce illumination bias.

The moving average(mean intensity) at step k + 1 is given by:
$$
m(k+1)=m(k)+\frac{1}{n}(z_{k+1}-z_{k-n})
$$
The threshold value is then set as:
$$
T_{xy}=bm_{xy}
$$
where b is some constant.


The algorithm is initialized only once, not at every row. In general, thresholding based on moving averages works well when the objects of interest are small (or thin) with respect to the image size, a condition satisfied by images of typed or handwritten text.

# Multivariable Thresholding
When the image acquisition sensors can make more than two variables(greyscale) available, multivariable thresholding is used to characterize pixels in say, the 3D space $Z^3$ ie, coloured images.

==The terminology used for 'pixels' in the 3D space is 'voxels' to represent _volumetric elements_==

One way to segment a colour image based on this parameter is to compute a distance measure, D(z, a), between an arbitrary colour point, z, and the average colour, a. Then, we segment the input image as follows:

$$
g=
\begin{cases}
1 \text{ if }D(z, a)<T\\
0 \text{ otherwise }
\end{cases}
$$
The equation D(z, a) = T describes a sphere (called a hypersphere) in n dimensional Euclidean space. A more powerful distance measure is the so-called Mahalanobis distance which is calculated using the covariance matrix of $z$s.

# Region-based segmentation

The following are segmentation techniques that based on directly finding the regions in an image:

## Region Growing
Region growing is a procedure that groups pixels or subregions into larger regions based on predefined criteria for growth. The basic approach is to start with a set of “seed” points and from these grow regions by appending to each seed those neighbouring pixels that have predefined properties similar to the seed such as specific ranges of intensity or colour. 

The initial seed choice depends on the availability of information. If none is available, we compute each pixel with the properties. However, if some basic pixel clusters are known, we could choose the seeds at the centroids of such clusters.

The selection of similarity criteria depends not only on the problem under consideration, but also on the type of image data available. For example, the analysis of land-use satellite imagery depends heavily on the use of colour. This problem would be significantly more difficult, or even impossible, to solve without the inherent information available in colour images. When the images are monochrome, region analysis must be carried out with a set of descriptors based on intensity levels and spatial properties such as moments or texture.

Another problem in region growing is the formulation of a stopping rule.
Region growth should stop when no more pixels satisfy the criteria for inclusion in that region. Criteria such as intensity values, texture, and colour are local in nature and do not take into account the “history” of region growth. Additional criteria that increase the power of a region-growing algorithm utilize the concept of size, likeness between a candidate pixel and the pixels grown so far (such as a comparison of the intensity of a candidate and the average intensity of the grown region), and the shape of the region being grown. The use
of these types of descriptors is based on the assumption that a model of expected results is at least partially available.

## Region Splitting and Merging
This technique attempts to subdivide an image initially into a set of arbitrary, disjoint regions and then merge and/or split the regions in an attempt to satisfy the conditions of segmentation.

A basic approach is to start with the entire image R, and recursively keep splitting the image into quadrants based on some predicate $Q(R_i)$= TRUE or FALSE. 

==This particular splitting technique has a convenient representation in the form of so-called quadtrees, that is, trees in which each node has exactly four descendants, with the root being the entire image.==

If only splitting is used, the final partition normally contains adjacent regions with identical properties. This drawback can be remedied by allowing merging as well as splitting. That is, two adjacent regions Rj and Rk are merged
only if $Q(R_j \cup R_k) = TRUE$.

It is customary to specify a minimum quad-region size beyond which no further splitting is carried out.

Properties based on the mean and standard deviation of pixel intensities in a region attempt to quantify the texture of the region. The concept of texture segmentation is based on using measures of texture in the predicates. In other words, we can perform texture segmentation by region splitting/merging simply by specifying predicates based on texture content.

# Segmentation Using Morphological Watersheds
Segmentation by watersheds combines the concepts of other segmentation techniques like edge detection, thresholding and region growing and as such is a more stable segmentation technique including properties like connected segmentation boundaries.

One of the principal applications of watershed segmentation is in the extraction of nearly uniform (bloblike) objects from the background. Regions characterized by small variations in intensity have small gradient values. Thus, in practice, we often see watershed segmentation applied to the gradient of an image, rather than to the image itself.

## Algorithm

Treat the grayscale image f(x,y) as a **3D surface**:

- Intensity → height
- Dark regions → valleys (basins)
- Bright regions → mountains (ridges)

Segmentation is done by **flooding the surface** from minima.
## Core Formulation

Let:

- $M_i$​ = regional minima
- $C(M_i)$ = catchment basin of minimum $M_i$

Then watershed segmentation partitions the image as:

$$
\bigcup_i C(M_i) = \text{image domain}, \quad C(M_i) \cap C(M_j) = \emptyset \ (i \ne j)
$$
Watershed lines (boundaries):

Watershed= complement of all catchment basins
## Algorithm (Conceptual Flooding)

1. Start from all **local minima**
2. Gradually “fill” basins with water
3. When water from different basins is about to merge:
    - Build a **dam (watershed line)**
4. Continue until entire image is flooded

## Marker-Based Watershed (Practical Form)

To avoid over-segmentation:

- Define markers:
    - $M_f$: foreground markers
    - $M_b$: background markers

Segmentation becomes:
$$\text{Watershed}( \nabla f \ \text{with markers} )$$
The gradient of the image is used in order to emphasize edges, since edges(with high intensity changes) will have high gradients.

## Core Insight

> Watershed finds boundaries by **simulating flooding from minima**, separating regions where water would naturally collect.

## Use Cases

### 1. Image Segmentation (Primary)

- Separates touching/overlapping objects
- Especially useful in:
    - Cell segmentation (medical images)
    - Industrial inspection

### 2. Edge-Based Segmentation

- Applied on gradient:

High gradient⇒barriers 

### 3. Marker-Controlled Segmentation

- Use prior knowledge to control regions
- Reduces over-segmentation

### 4. Object Separation

- Splits connected components based on topology
- Works well when shapes touch

## Key Advantage

- Produces **closed, well-defined boundaries**
- Naturally handles **region-based segmentation**

## Key Limitation

- **Over-segmentation** (too many regions)
- Solved using:
    - Smoothing
    - Marker-based approach

## One-Line Summary

> Watershed segmentation treats the image as a surface and **partitions it into regions by flooding from minima, forming boundaries where regions meet**.

### Why Markers Are Needed

The basic watershed uses **all local minima** → this creates **severe over-segmentation** (too many tiny regions due to noise and texture).

**Markers solve this by controlling where flooding starts.**

## Core Idea

Instead of flooding from _every_ minimum:

- Define a **small set of meaningful seeds (markers)**
- Flood only from these markers
- Ignore all other minima

## Types of Markers

- **Foreground markers ($M_f$​)**  
    → pixels definitely inside objects
- **Background markers ($M_b$)**  
    → pixels definitely outside objects

## Modified Formulation

Watershed is applied on a modified image (usually gradient):
$$
\text{Watershed}( \nabla f \ \text{with imposed minima at markers} )
$$

This means:

- Only marker locations act as **valid minima**
- All other minima are suppressed

## Conceptual Effect

- Each marker becomes a **source of flooding**
- Regions grow outward from markers
- Boundaries form where growing regions meet

So segmentation becomes:

> “Assign every pixel to the nearest marker region”

## How Markers Are Created

### 1. Manual

- User selects regions (rare in automation)

### 2. Automatic (common)

Examples:

- Thresholding
- Distance transform
- Morphological operations (opening, closing)

## Key Benefits

### 1. Eliminates Over-Segmentation

- Reduces number of regions drastically

### 2. Incorporates Prior Knowledge

- You control what counts as an object

### 3. Produces Meaningful Regions

- Each region corresponds to a **real object**

## Intuition

Without markers:

- “Flood from every tiny dip → chaos”

With markers:

- “Flood only from important locations → controlled segmentation”

## One-Line Summary

> Markers guide the watershed by **restricting flooding to selected seed points**, eliminating over-segmentation and producing meaningful object-based regions.


# Use of Motion
Motion-based segmentation detects moving objects by analyzing changes across consecutive image frames. The main assumption is that backgrounds are static, ie, they remain almost unchanged while foreground components produce intensity changes over time.

Comparing the same pixel at different instants of time, we have
- Reference frame: $f(x,y,t_i)$
- Current frame: $f(x,y,t_j)$
Then the difference calculated to detect motion is:
$$
d_{ij}(x,y)=
\begin{cases}
1 & \text{if } |f(x,y,t_i)-f(x,y,t_j)| > T \\
0 & \text{otherwise}
\end{cases}
$$
## Accumulative Difference Images (ADI)
Instead of comparing only two frames, motion can be accumulated over many frames.

Suppose:

$$f(x,y,t_1), f(x,y,t_2), \dots, f(x,y,t_n)$$

with reference image:

$$R(x,y)=f(x,y,t_1)$$

The ADI counts how many times each pixel changes over the sequence.

1. Absolute ADI:
$$
A_k(x,y)=
\begin{cases}
A_{k-1}(x,y)+1 & \text{if } |R(x,y)-f(x,y,k)|>T \\
A_{k-1}(x,y) & \text{otherwise}
\end{cases}
$$
Counts all motion occurrences regardless of direction/intensity change and is useful for:
- detecting motion paths
- estimating object trajectory
- identifying frequently moving regions

2. Positive ADI:
$$
P_k(x,y)=
\begin{cases}
P_{k-1}(x,y)+1 & \text{if } [R(x,y)-f(x,y,k)]>T \\
P_{k-1}(x,y) & \text{otherwise}
\end{cases}
$$
Tracks pixels becoming darker/lighter depending on object-background relationship and is useful for:
- locating initial object position
- determining displacement

3. Negative ADI:
$$
N_k(x,y)=
\begin{cases}
N_{k-1}(x,y)+1 & \text{if } [R(x,y)-f(x,y,k)]<-T \\
N_{k-1}(x,y) & \text{otherwise}
\end{cases}
$$

In general,
- Positive ADI: often marks original object location
- Negative ADI: often marks new object location
- Absolute ADI: combines both

From these parameters, we can estimate the direction, speed and trajectory

## Frequency Domain Techniques

Instead of directly comparing pixels, motion can be analyzed using Fourier transforms.

The core idea is that moving objects generate periodic changes over time. These temporal changes appear as frequency peaks in the Fourier spectrum.

From a sequence of frames, f(x, y, t), we can obtain weighted projections along the X and Y axes as $g_x(t,a_1)$ and $g_y(t,a_2)$ and through the temporal Fourier Transforms, $G_x(u_1,a_1)$ and $G_y(u_2,a_2)$

Velocity components can then be obtained from the spectral peaks: $u_1=a_1V_1$ and $u_2=a_2V_2$.

The overall interpretation is that:
- Static background  
    → produces DC component (frequency 0)
- Moving objects  
    → produce peaks at nonzero frequencies

Peak locations thus reveal object speed and motion direction.

Advantages of Frequency Domain Motion Analysis are as follows:

1. Detects Very Small Motion
2. Robust Against Noise
3. Direct Velocity Estimation
4. Useful for Tracking

Common in:

- radar
- satellite imaging
- surveillance
- remote sensing

The main applications are:

 Spatial Methods:
- Surveillance
- Traffic monitoring
- Human tracking
- Background subtraction

Frequency Methods:
- Radar tracking
- Satellite imaging
- Remote sensing
- Motion estimation in noisy environments
