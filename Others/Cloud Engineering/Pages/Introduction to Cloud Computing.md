#personal 

Cloud Computing is the on-demand delivery of computing resources on a pay-as-you-go pricing ranging from applications to data centres. Examples of computing resources are networks, servers, storage and applications.

Key characteristics of the cloud are Self-service on-demand, broad network access, resource pooling, fast elasticity and measurable service.

**Data Centers** refer to the locations that provides the facilities required for virtualization on a certain scale, eg, servers, coolers, electricity and power etc. Their major flaws were commitment and continuous costs. This was solved by the pay-as-you-go property of cloud computing besides the need to maintain one's own data centres.

# Types of Deployment Model

## Private Cloud
Data of a single organization is stored and the data centre lies within the organization. It can be expensive as all hardware and individuals for managing the cloud must be provided by the organization.

## Public Cloud
A cloud vendor maintains a public cloud which can be accessed by the public. Costs are reduced

## Hybrid Cloud
A combination of private, community and public cloud. It provides enhanced security and performance. 

# Reliability and Availability

An **Availability Zone** is a collection of one or more data centres that can act as a failure zone.

A **Region** is the geographical location of data centres. Each region has two or more availability zones.

# Benefits of the Cloud

1. **On-demand** self-service of computing power, storage, networks, software etc. This provides scalability on runtime and also implements the pay-as-you-go structure.
2. **Broad Network Access** through devices like laptops, tablets etc. Raises security concerns mitigated through use of private cloud.
3. **Resource pooling** provides immediately available "infinite" scale of resources like storage, bandwidth etc. Implements "rapid elasticity".
4. **Rapid Elasticity** refers to the ability of the cloud to scale rapidly according to needs, whether it be increasing or reducing resources.
5. **Measured service** refers to the observation of services provided by the vendor for reasons like billing, planning and effective utilization.
6. Shifting to cloud leads to **reduced IT costs** by cutting the need to manage IT systems, upgrades, expensive equipment etc.
7. **Scalability** allows the services to scale up or down
8. **Business Continuity** protects the data from natural disasters, power failure and other crisis which might otherwise reduce productivity.
9. **Collaboration efficiency** by allowing the sharing of resources across different geographical regions.


# Risks of the Cloud

1. Measuring the cost of owning their own equipment vs leasing, considering factors like performance, security etc.
2. Compatibility with the present IT structure and business policies.
3. Trust on various service providers for events like crisis and downtime.
4. Confidentiality of sensitive data.
5. Lack of control over performance due to quality of service of the vendor.
